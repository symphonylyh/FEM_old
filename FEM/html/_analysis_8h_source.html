16 /* Abstract base Analysis class with shared public methods and pure virtual methods.
17  *
18  * Key concepts of during the design of this class:
19  *
20  * 1. One of the benefits of C++ inheritance feature is the code can be kept
21  * in the most concise way. For example, we defined a Analysis base class here,
22  * and we will have derived class such as linear elastic class, nonlinear elastic,
23  * viscoelastic, etc. Furthermore, we can also have classes derived from the
24  * already derived class, like linear isotropic and linear anisotropic class from
25  * the linear elastic class. The subroutines/functions/procedures that are
26  * shared by the derived classes can be abstracted and defined in the base class,
27  * while derived classes can still define other subroutines/functions/procedures
28  * as their own features. In this way, our program is elegantly organized and
29  * extendable, with the minimum redundancy in codes.
30  *
31  * 2. The information such as mesh, global stiffness, force, etc should be used
32  * in any type of the problems, so they have been made public (specifically,
33  * protected) in Analysis class. By inheritance we have all those info as public
34  * in the derived classes. It is noted that we should avoid name confliction
35  * when we define the own variables of the derived classes.
36  * In addition, from a memory allocation view, the base class's memory space is
37  * allocated inside the memory space of this derived class. But these space are
38  * just copies from the base class, i.e., if we have a second derived class,
39  * these space are independent and uniquely of their own.
40  */
