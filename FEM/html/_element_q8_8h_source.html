21  /* Private helper structure for the bullet-proof memory management of
22  * the static members of this class.
23  *
24  * This is the elegant solution to pave our path for a powerful
25  * hybrid element system. There are several improvements made throughout
26  * the development of this module:
27  *
28  * 1. "Shape" is a general property of a given element type, which does
29  * not depend on the instances of that class. In other words, if we
30  * add Shape as a class member, we will have 100 Shape objects in memory when
31  * we create 100 elements, which is very unwise. Thus, we use a static
32  * member that is a intrinsic property of the whole class and can be
33  * shared by all its instances (with memory just allocated once).
34  *
35  * 2. At first we define a static Shape object in the class as:
36  *
37  * in .h: static ShapeQ8 shape;
38  * in .cpp: ShapeQ8 ElementQ8::shape;
39  *
40  * However, when we want to use the shape in our hybrid system (for
41  * apply load, strain and stress computation, etc), it is inconvenient
42  * that we need to figure out which type the element is and the choose
43  * the corresponding shape. This requires a if/else judge and becomes
44  * redundant when element types increase. Thus, a polymorphed Shape
45  * pointer is used to provide the flexibility.
46  *
47  * 3. So far so good. But, in order to use a pointer we need to allocate
48  * it on heap by Shape* shape = new ShapeQ8, then where can we delete it
49  * to prevent memory leak? One option is to delete this static pointer
50  * before we exit the main(), but this means this class is not
51  * self-consistent and requires the client to be careful enough to delete
52  * it at the end, which no one can promise. Thus, we use an internal
53  * structure to manage all the static members we have in the class, and
54  * the constructor & destructor of this structure takes over the
55  * create & destroy task for us. Before the program starts, the static
56  * structure is initialized by its constructor; when the program exists,
57  * the structure is destroyed and its destructor is called so that the
58  * shape object on heap is deallocated. This is a perfect solution for
59  * memory management.
60  */
61  struct staticMembers {
62  Shape* shape; // we can have more static members, if needed
63

68  shape = new ShapeQ8;
69  }
